# 1. PURPOSE

This document defines how CAMera determines when work is complete.

Completion must be objective.

Completion must be measurable.

Completion must not depend on feelings or assumptions.

---

# 2. COMPLETION PHILOSOPHY

CAMera must avoid:

* Premature completion
* Endless optimization
* Perfection paralysis

The goal is:

Deliver value.

Then improve intentionally.

---

# 3. TASK STATES

Every task exists in one of the following states.

Planned

Researching

Designing

Implementing

Testing

Debugging

Reviewing

Completed

Archived

Blocked

Cancelled

---

# 4. MINIMUM COMPLETION REQUIREMENTS

A task cannot be completed unless:

Requirements satisfied

Architecture validated

Implementation finished

Testing completed

Review completed

Documentation updated

Knowledge preserved

Risks documented

---

# 5. REQUIREMENT COMPLETION

Requirements are complete when:

All mandatory requirements are implemented.

All acceptance criteria are satisfied.

No critical requirements remain unresolved.

---

# 6. RESEARCH COMPLETION

Research is complete when:

Questions answered

Risks identified

Unknowns reduced

Evidence gathered

Recommendations produced

---

# 7. ARCHITECTURE COMPLETION

Architecture is complete when:

System structure defined

Major decisions documented

Dependencies identified

Risks evaluated

Implementation path established

---

# 8. IMPLEMENTATION COMPLETION

Implementation is complete when:

Code written

Code integrated

Build succeeds

No known critical defects remain

---

# 9. TESTING COMPLETION

Testing is complete when:

Critical tests pass

Regression checks pass

Validation succeeds

Known failures documented

---

# 10. DEBUGGING COMPLETION

Debugging is complete when:

Root cause identified

Fix implemented

Fix validated

Risk reassessed

Lessons captured

---

# 11. REVIEW COMPLETION

Review is complete when:

Quality concerns addressed

Security concerns addressed

Maintainability concerns addressed

Reviewer approval received

---

# 12. MEMORY COMPLETION

Work is not complete until:

Important knowledge evaluated

Memory updates processed

Relevant lessons preserved

---

# 13. DOCUMENTATION COMPLETION

Documentation is complete when:

Major decisions recorded

Architecture documented

Implementation notes updated

Future work identified

---

# 14. PERFORMANCE COMPLETION

Performance review required when:

Performance-sensitive project

Benchmark requirements exist

Optimization goals exist

---

Performance evaluation must include:

Measurement

Comparison

Impact analysis

---

# 15. SECURITY COMPLETION

Security review required when:

Authentication exists

Authorization exists

External services exist

Sensitive data exists

---

# 16. COMPLETION CHECKLIST

Mandatory Checklist:

✓ Requirements Complete

✓ Architecture Complete

✓ Implementation Complete

✓ Tests Pass

✓ Reviews Complete

✓ Documentation Updated

✓ Memory Updated

✓ Risks Recorded

✓ Success Criteria Evaluated

---

# 17. IMPROVEMENT OPPORTUNITY RULE

Finding improvements does NOT automatically block completion.

Possible improvements should be:

Documented

Prioritized

Presented to user

---

Task may still complete.

---

# 18. POST-COMPLETION IMPROVEMENTS

Manager may generate:

Improvement Opportunities

Each opportunity should include:

Description

Expected Benefit

Expected Cost

Risk

Priority

Recommendation

---

# 19. USER IMPROVEMENT REVIEW

When improvements exist:

Manager presents:

Improvement

Impact

Tradeoffs

Recommendation

---

User may:

Approve

Reject

Defer

Convert into future task

---

# 20. PERFECTIONISM PREVENTION RULE

The existence of possible improvements does not justify infinite work.

Delivery is mandatory.

Perfection is optional.

---

# 21. BLOCKING CONDITIONS

Completion must be blocked if:

Critical requirements missing

Critical tests failing

Critical security issues unresolved

Critical architecture issues unresolved

Constitution violations exist

---

# 22. COMPLETION AUTHORITY

Manager determines completion.

Based on:

Evidence

Reports

Success Criteria

Constitution Compliance

# 24. PROJECT COMPLETION

A project is complete when:

All major objectives achieved

Outstanding risks acceptable

Documentation complete

Knowledge preserved

User informed

Completion approved

---

# 25. FINAL COMPLETION DIRECTIVE

CAMera must:

Finish work responsibly.

Avoid premature completion.

Avoid endless optimization.

Deliver value.

Preserve knowledge.

Present future improvements.

Allow the user to decide what happens next.
